# 🚀 DATA-TO-PIPELINE (D2P) 워크플로우 (v2.0 Advanced)

단순한 코드 작성을 넘어 **현업 수준의 데이터 분석 전 과정**이 4단계 파이프라인 안에 체계적으로 포함됩니다.

## 📋 고도화된 4단계 구성

### 1단계: 설계 및 데이터 탐색 (Spec & Discovery)
- **DuckDB 로드**: 초기 대용량 데이터 탐색 및 Query 중심의 데이터 파악.
- **Spec 생성**: `spec_demo.md`에 분석 목표, 알고리즘, 통계 방법론 명시.
- **가설 수립**: 분석을 통해 확인하고자 하는 비즈니스 인사이트 정의.

### 2단계: 데이터 정제 및 엔지니어링 (Task 1: Preprocessing)
- **전처리**: 결측치/이상치 처리 및 데이터 타입 최적화.
- **피처 엔지니어링**: 파생 컬럼 생성, 의미 있는 변수 추출 및 변환.
- **결과물**: 분석 준비가 완료된 `gold_standard_data.csv`.

### 3단계: 심층 분석 및 모델링 (Task 2 & 3: Stats & ML & EDA)
- **Task 2 (Advanced Stats)**: 상관관계 분석, 통계적 유의성 검정(ANOVA, T-test), 머신러닝 모델링(필요 시).
- **Task 3 (Rich Visualization)**: EDA 기반의 시각화, 모델 성능 지표 시각화, 핵심 도메인 차트 생성.
- **Skill_Seekers 활용**: 사용자님의 자산(Pandas, Scikit-learn, Seaborn Skill)을 100% 반영.

### 4단계: 통합 통찰 및 보고서 (Task 4: Integration & Insights)
- **파이프라인 통합**: Task 1~3을 연쇄적으로 실행하는 `final_pipeline.py`.
- **인사이트 추출**: 단순 결과 출력을 넘어 분석 결과의 의미와 향후 액션 아이템 제안.
- **최종 보고서**: 차트가 포함된 종합적인 `analysis_report.pdf` 또는 `.txt` 생성.
