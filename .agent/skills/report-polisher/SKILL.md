# 🕵️ Report Polisher Protocol

이 스킬은 사용자의 분석 리포트(Markdown/Text)를 **McKinsey/Bain 스타일의 전문 컨설팅 보고서**로 업그레이드할 때 사용한다.

## 1. 🏗️ 구조화 원칙 (Pyramid Structure)
*   **BLUF (Bottom Line Up Front)**: 모든 섹션의 첫 문장은 해당 단락의 **핵심 결론**이어야 한다. "배경 -> 분석 -> 결론"이 아니라 "**결론 -> 근거(데이터) -> 제언**" 순서를 강제한다.
*   **MECE (상호 배제, 전체 포괄)**: 분석 내용이 겹치거나(Overlap) 누락(Gap)되지 않도록 논리 구조를 재배치한다.

## 2. ✍️ 문체 가이드 (Tone & Manner)
*   **No Ambiguity**: "많은 것 같다", "꽤 높다" 등 모호한 표현 금지. -> "**전월 대비 43% 상승했다**", "**상위 10%에 해당한다**" 등 수치 중심으로 변경.
*   **Active Voice**: "분석되었습니다" (수동태) -> "**확인했다/도출했다**" (능동태) 또는 "**나타났다**" (객관적 서술).
*   **Formatting**: 줄글(Text Block)은 반드시 **불렛 포인트(Bullet Points)** 또는 **표(Table)**로 시각화한다.

## 3. 🚦 실행 프로세스
1.  **Read**: 대상 파일(`TargetFile`)을 읽어 논리적 허점과 가독성 문제를 진단한다.
2.  **Critique**: "Before vs After" 예시를 보여주고 수정 방향을 제안한다.
3.  **Rewrite**: 사용자의 승인을 얻어 전체 문서를 윤문(Refining)한다.

## 4. 📝 예시 (Example)
*   *Before:* "텀블러 가격이 브랜드가 있으면 비싼 것 같다. 스탠리 같은 게 인기가 많아서 그런 듯."
*   *After:* "**브랜드 파워와 가격의 양의 상관관계 확인 (+0.63)**. 특히 '스탠리' 키워드 포함 시 평균 가격이 타 제품 대비 **1.5배** 높게 형성됨을 데이터로 입증."
