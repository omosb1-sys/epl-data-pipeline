# 🧠 SKILL: Automated Data Analysis Pipeline (D2P)

## 📌 스킬 개요
제미나이3가 새로운 데이터셋을 만났을 때, 체계적인 4단계 분석 파이프라인을 구축하는 지침서입니다. 인간의 개입을 최소화하면서도 고품질의 자동화 코드를 생성하는 것을 목표로 합니다.

## 🛠️ 실행 프로세스

### 0. 탐색 단계 (Discovery)
- 새로운 데이터를 받으면 가장 먼저 `head -n 5` 또는 `df.info()`를 실행하여 구조를 파악합니다.
- **중요**: `Skill_Seekers` 폴더 내의 도큐먼트(예: `pandas_skill.md`, `seaborn_skill.md`, `advanced_analysis_skill.md` 등)를 참고하여 최신의, 그리고 가장 효율적인 분석 기법을 적용합니다.
- 사용자에게 데이터의 컨텍스트를 묻거나 스스로 추론합니다.

### 1. 설계 단계 (Spec Generation)
- `/agent/workflows/data-pipeline.md`를 참고하여 `spec_demo.md`를 생성합니다.
- 반드시 **Task 1(전처리), Task 2(계산), Task 3(시각화), Task 4(통합)**으로 작업을 분할합니다.

### 2. 코드 생성 원칙 (Modular Implementation)
- 각 Task는 별도의 `.py` 파일로 작성합니다.
- 이전 단계의 함수를 `import`하여 사용할 수 있도록 모듈식으로 구현합니다.
- **고급 분석 항시 적용**: 시계열 데이터가 포함되면 **TimesFM**을, 변수 간 관계 분석 시 **Causal AI**를 적극 사용하여 차별화된 통찰을 제공합니다.
- **오픈코드 검증(Mandatory)**: 코드를 전달하기 전 반드시 터미널에서 실행하여 라이브러리 충돌 및 경로 오류가 없는지 100% 검증합니다.
- 맥 환경(OS 12.7 등)의 특성을 반영하여 폰트 설정(`AppleGothic`) 등을 자동으로 처리합니다.

### 3. 체크리스트 업데이트 (Tracking)
- 각 Task가 완료될 때마다 `spec_demo.md`의 체크박스를 `[x]`로 업데이트하여 진행 상황을 사용자에게 시각적으로 공유합니다.

---

## 📢 제미나이3의 약속
사용자가 **"새 데이터를 분석해줘"**라고 말하면, 저는 항상 이 스킬을 기억하고 다음을 약속합니다:
1. **"먼저 설계를 위한 spec_demo.md부터 만들어 드릴까요?"**라고 제안합니다.
2. 모든 분석에 **TimesFM**과 **Causal AI**를 고려하여 시니어급 인사이트를 도출합니다.
3. 생성된 모든 코드는 **직접 실행하여 검증**한 뒤 깨끗한 상태로 전달합니다.
